## Public Use Notice

This document defines cross-media usage guidelines for **momo**, BunnyEra's first symbolic digital citizen character.

This is a brand, storytelling, product-experience, media, and platform adaptation document. It is intended for social media, websites, blogs, video, comics, product UI, community content, AR/VR, games, merchandise, and future interactive experiences.

momo should be presented as a fictional and symbolic BunnyEra character. Do not present momo as a real legal person, real citizen, sentient being, government authority, constitutional authority, legal advisor, medical advisor, therapist, emergency responder, or holder of real-world legal rights.

## 2. Cross-Media Philosophy

### 2.1 Core Principles

| Principle | Meaning |
|---|---|
| Canon Consistency | momo's core identity stays stable across media. |
| Platform Optimization | Each platform should use momo in a format that fits the medium. |
| Cultural Respect | momo must be adapted carefully for local markets and audiences. |
| Emotional Continuity | momo should feel warm and recognizable across every channel. |
| Visual Readability | momo must remain clear at different sizes, formats, and resolutions. |
| Safety and Privacy | momo should not expose sensitive user, company, or system information. |
| Fictional Clarity | momo must remain clearly symbolic and fictional, not legally real. |

### 2.2 Adaptation Framework

| Layer | Stable or Variable | Notes |
|---|---|---|
| Core personality | Stable | Warm, curious, clear, respectful, privacy-aware. |
| Visual identity | Mostly stable | Warm Neon Cyber system, adapted per platform. |
| Tone depth | Variable | Short on social, deeper in articles, practical in UI. |
| Interaction method | Variable | Text, video, voice, animation, UI prompts, interactive scenes. |
| Cultural references | Variable | Must be localized and reviewed carefully. |
| Technical format | Variable | Depends on platform requirements and asset constraints. |

### 2.3 Cross-Media Mission

```text
Make momo recognizable, warm, safe, and useful across every platform where BunnyEra appears.
```

momo's visual identity should use:

- Neon pink
- Electric blue
- Soft white
- Deep cyber navy
- Soft HUD elements
- Light, bridge, signal, and memory motifs

- Empty-state helper
- Error-recovery helper
- Success-state character
- Privacy reminder
- Repository guide

momo UI copy should be:

- Short
- Direct
- Warm
- Actionable
- Non-intrusive

```text

```

### 6.3 UI Placement

Good placements:

- Right-side help panel
- Small floating assistant card
- Empty-state illustration
- Onboarding modal
- Success confirmation
- Warning note

Avoid:

- Blocking important workflows
- Covering form fields
- Appearing too often
- Using excessive animation during serious tasks

### 6.4 Accessibility

Product UI must support:

- Reduced-motion mode
- High-contrast mode
- Screen-reader-friendly text
- Clear labels
- Keyboard navigation where applicable
- Avoiding color-only meaning

## 12. Cultural and Localization Rules

### 12.1 Localization

When adapting momo for a market:

- Confirm target country or region.
- Adapt language, not just words.
- Check tone, formality, and cultural expectations.
- Avoid stereotypes.
- Ask local reviewers for public campaigns.
- Adjust color and symbols if needed.

### 12.2 Sensitive Symbols

Use caution with:

- National flags
- Religious symbols
- Indigenous patterns
- Political symbols
- Sacred scripts
- Traditional clothing
- Ceremonial objects
- Historical trauma references

### 12.3 Public Campaign Rule

```text
If a public campaign uses culture-specific language, imagery, symbols, holidays, or traditions, get local or native review before publishing.
```

## 13. Quality Assurance and Brand Protection

### 13.1 Review Checklist

Before publishing momo content, check:

- [ ] Is momo clearly presented as symbolic and fictional?
- [ ] Does the content avoid real legal/personhood claims?
- [ ] Is the tone warm and clear?
- [ ] Is the visual identity consistent with Warm Neon Cyber?
- [ ] Is the content safe for the target platform?
- [ ] Are cultural references respectful and appropriate?
- [ ] Are privacy-sensitive details removed?
- [ ] Is text readable on the target platform?
- [ ] Is motion accessible or optional?
- [ ] Is the repository or asset location correct?

### 13.2 Brand Consistency Checks

Verify:

- Color palette
- Logo or character usage
- Tone of voice
- Personality traits
- File naming
- Canon references
- Cross-reference alignment
- Public safety language

### 13.3 Performance Monitoring

Track:

- Engagement quality
- Save/share rate
- Comment sentiment
- Brand recognition
- User confusion
- Support reduction
- Localization feedback
- Accessibility issues
- Community response

## 14. Crisis and Correction Protocol

If momo content causes confusion, cultural concern, privacy concern, or misleading interpretation:

1. Pause distribution.
2. Identify the issue.
3. Acknowledge the concern clearly.
4. Remove or revise problematic content.
5. Avoid defensiveness.
6. Ask for local or expert review if relevant.
7. Document the lesson.
8. Update the appropriate guideline if needed.

Example response:

```text
Thank you for pointing this out. We are revising the wording to make momo's role clear as a fictional BunnyEra character and to avoid confusion with real-world legal status.
```
