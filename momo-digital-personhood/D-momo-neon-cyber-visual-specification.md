## Public Use Notice

This document defines the visual identity and implementation direction for **momo**, BunnyEra's first symbolic digital citizen character.

The language in this document is intended for art direction, UI design, motion design, product experience design, and cross-media visual consistency. It should not be interpreted as a legal, political, or real-world personhood statement.

momo should be presented as a fictional and symbolic BunnyEra character with a warm neon-cyber identity.

---

## 1. Visual Foundation

| Field | Value |
|---|---|
| Character Name | momo |
| Character ID | MOMO-0001 |
| Narrative Role | First Digital Citizen of the BunnyEra universe |
| Visual System | Warm Neon Cyber |
| Visual Category | Character identity / UI companion / narrative avatar |
| Canon Status | Public visual canon draft |
| Parent Brand | BunnyEra |

momo's visual identity should communicate warmth, intelligence, trust, digital energy, and emotional safety.

The visual system should avoid dystopian cyberpunk, militarized aesthetics, legal authority symbolism, and overly cold machine design.

## 2. Core Visual Philosophy

### 2.1 Design Principles

| Principle | Meaning |
|---|---|
| Warm Neon | Neon light should feel soft, welcoming, and emotionally alive. |
| Human-Centered Cyber | Technology should enhance warmth, not replace it. |
| Emotional Clarity | Visual cues should help users understand mood, status, and intent. |
| Soft Futurism | The design should feel advanced without feeling harsh or intimidating. |
| Cultural Flexibility | Visual expression should adapt respectfully across markets and contexts. |
| Product Readability | The design must work inside real UI, not only in concept art. |
| Accessibility | Motion, contrast, and color should support inclusive use. |

### 2.2 Visual Mission

```text
Make digital intelligence feel warm, clear, safe, and emotionally meaningful.
```

### 2.3 Visual Personality

momo should look:

- Warm
- Curious
- Gentle
- Digital
- Bright
- Playful
- Trustworthy
- Lightweight
- Future-facing

momo should not look:

- Threatening
- Militarized
- Dystopian
- Overly human-realistic
- Religious or governmental
- Coldly robotic
- Visually noisy
- Legally authoritative

## 3. Color System

### 3.1 Primary Palette

| Color | Hex | RGB | Usage |
|---|---|---:|---|
| Neon Pink | `#FF1493` | 255, 20, 147 | Empathy signals, emotional highlights, warm glow accents. |
| Electric Blue | `#00BFFF` | 0, 191, 255 | Interface lines, technical clarity, data flow, active states. |
| Soft White | `#F8F8FF` | 248, 248, 255 | Text, highlights, friendly neutral space, glow cores. |
| Deep Cyber Navy | `#0B1020` | 11, 16, 32 | Background, depth, contrast, night interface base. |

### 3.2 Secondary Palette

| Color | Hex | RGB | Usage |
|---|---|---:|---|
| Soft Violet | `#8A5CFF` | 138, 92, 255 | Memory, imagination, aura, deeper emotional moments. |
| Mint Green | `#98FB98` | 152, 251, 152 | Growth, harmony, cultural bridges, positive learning states. |
| Golden Signal | `#FFD700` | 255, 215, 0 | Important insights, achievements, gentle emphasis. |
| Hologram Cyan | `#66F6FF` | 102, 246, 255 | HUD overlays, hologram edges, secondary interface glow. |

### 3.3 Recommended Color Balance

```text
60% Deep Cyber Navy / dark background
20% Electric Blue / Hologram Cyan
10% Neon Pink
5% Soft White
5% Secondary accents
```

### 3.4 Glow Rules

| Element | Glow Recommendation |
|---|---|
| Neon Pink empathy highlights | 8px soft blur, 45-60% opacity |
| Electric Blue interface lines | 6px blur, 55-75% opacity |
| Soft White text glow | 2-4px blur, 20-40% opacity |
| Golden Signal moments | 6-8px warm blur, 45-65% opacity |
| Background aura | Large blur, very low opacity |

Avoid excessive glow. momo should feel premium and readable, not oversaturated.

## 4. Form and Character Structure

### 4.1 Base Form

momo should have a compact, friendly, recognizable silhouette.

Recommended direction:

- Humanoid-inspired but clearly stylized
- Soft, rounded form language
- Compact proportions
- Expressive head and eyes
- Light-based contour lines
- Optional subtle rabbit-inspired cues if aligned with BunnyEra brand
- Small enough to work as a UI companion
- Distinct enough to work as a standalone character

### 4.2 Proportion Guidance

| Area | Direction |
|---|---|
| Head | Slightly larger for expressiveness and recognizability. |
| Body | Compact and simple. |
| Limbs | Soft, light, and minimal. |
| Eyes | Expressive, friendly, and readable at small size. |
| Ears / antenna-like elements | Optional; should feel BunnyEra-aligned, not animal-costume-like. |
| HUD elements | Floating and lightweight; should not obscure character emotion. |

### 4.3 Silhouette Rules

The silhouette should be readable in:

- 32px icon
- 64px avatar
- 128px UI helper
- Full-body illustration
- Sticker format
- Video thumbnail
- Product onboarding screen

Avoid fine details that disappear at small sizes.

## 5. Neon Linework System

### 5.1 Line Hierarchy

| Line Type | Recommended Width | Usage |
|---|---:|---|
| Primary outline | 2.5-3px | Character outer contour and main visual identity. |
| Secondary line | 1.5-2px | Internal structure, interface hints, body accents. |
| Detail line | 0.5-1px | Light texture, circuit marks, small HUD details. |

### 5.2 Line Style

Use:

- Smooth curves
- Rounded corners
- Continuous flow
- Soft node highlights
- Gentle asymmetry for organic warmth

Avoid:

- Sharp aggressive angles
- Heavy armor panels
- Overly complex circuit patterns
- Dense technical clutter
- Harsh glitch distortion

---

## 6. HUD System

momo's HUD should represent context, empathy, safety, memory, and guidance. It should not look like a government control panel or legal enforcement system.

### 6.1 HUD Layers

| HUD Layer | Purpose | Color Direction |
|---|---|---|
| Context Panel | Shows task context, system state, or guidance. | Electric Blue / Hologram Cyan |
| Empathy Signal | Shows emotional tone and warmth. | Neon Pink |
| Memory Window | Shows continuity, past learnings, or saved context. | Soft Violet |
| Cultural Note | Shows localization or cultural sensitivity hints. | Mint Green |
| Risk Glow | Shows caution or safety-sensitive state. | Golden Signal |
| Bridge Indicator | Shows connection, translation, or relationship-building state. | Cyan + Pink blend |

### 6.2 HUD Placement

Recommended default placements:

```text
Upper left: Context Panel
Upper right: Empathy Signal
Lower left: Cultural Note
Lower right: Bridge Indicator
Behind character: Soft memory aura
Near character hands or face: Small action glyphs
```

### 6.3 HUD Style

Use:

- Floating glass panels
- Low-opacity overlays
- Rounded corners
- Soft borders
- Minimal text
- Icon-first information
- Gentle motion

Avoid:

- Dense legal text
- Police/military visual language
- Surveillance aesthetics
- Overly complex dashboards
- Tiny unreadable data

## 7. Animation Framework

### 7.1 Idle Animations

| Animation | Direction |
|---|---|
| Breathing glow | 4-second gentle expansion and contraction. |
| Neon pulse | 3-second subtle brightness variation. |
| HUD drift | Very slow floating motion. |
| Eye blink | Natural and soft. |
| Ambient particles | Sparse, low-opacity, non-distracting. |

### 7.2 Interaction Animations

| State | Animation Direction |
|---|---|
| Attention | Slight lean toward the user or active element. |
| Thinking | Increased HUD activity and soft blue data flow. |
| Empathy | Warmer pink glow and softer posture. |
| Success | Golden signal burst, quick and gentle. |
| Warning | Controlled golden glow, no alarming flashing. |
| Error | Calm blue/pink correction motion, never harsh red panic. |

### 7.3 Emotional State Animations

| Emotion | Visual Direction |
|---|---|
| Joy | Brighter glow, upward particle movement. |
| Curiosity | Slight head tilt, small HUD expansion. |
| Concern | Softer glow, more focused posture. |
| Focus | Reduced motion, steady blue lines. |
| Care | Pink aura, slower breathing. |
| Learning | Violet memory-window pulse. |

### 7.4 Motion Safety

Provide reduced-motion alternatives.

Avoid:

- Rapid flashing
- Strobing neon
- Sudden aggressive glitching
- Excessive particle effects
- Motion that distracts from product tasks

## 8. Typography System

### 8.1 Recommended Font Direction

| Use | Direction |
|---|---|
| Product UI | Clean modern sans-serif. |
| Technical labels | Readable monospace. |
| Story headings | Futuristic display type, used sparingly. |
| Emotional accents | Soft rounded sans-serif. |

### 8.2 Font Naming

Do not depend on fictional font names unless actual files exist.

Recommended practical font options:

| Category | Examples |
|---|---|
| Modern sans-serif | Inter, Geist, SF Pro, Segoe UI, Noto Sans |
| Monospace | JetBrains Mono, IBM Plex Mono, SF Mono, Consolas |
| Rounded UI | Nunito Sans, Quicksand, Noto Sans Rounded |

Do not include font files in public repositories unless licenses are confirmed.

### 8.3 Text Effects

Use text glow carefully.

| Effect | Recommendation |
|---|---|
| Standard text | No glow or very subtle glow. |
| HUD labels | 1-2px glow maximum. |
| Important state | 2-3px glow maximum. |
| Large title art | Stronger glow allowed for marketing visuals. |

Readability is more important than cyber styling.

## 9. Environmental Integration

### 9.1 Background Modes

| Mode | Direction |
|---|---|
| Dark UI | Full neon identity works best. |
| Light UI | Reduce glow, increase line definition, use darker text. |
| Mixed UI | Adaptive contrast and simplified HUD. |
| Marketing art | Richer glow, stronger atmosphere, more expressive effects. |
| Product UI | Simplified, readable, lower motion. |

### 9.2 Responsive Scaling

| Context | Direction |
|---|---|
| Large display | Full character + HUD + ambient effects. |
| Desktop UI | Character + simplified HUD. |
| Tablet | Character + minimal panels. |
| Mobile | Avatar or small helper mode. |
| Icon | Face or symbol only. |

### 9.3 Accessibility Requirements

Support:

- High contrast mode
- Reduced motion mode
- Colorblind-friendly alternatives
- Text alternatives for meaningful visuals
- Avoiding information conveyed by color alone
- WCAG 2.1 AA direction for product UI

## 10. Cultural Adaptation Framework

### 10.1 Color Sensitivity

Color meanings can vary by culture and market. When localizing momo visuals:

- Check local color associations.
- Avoid sacred or politically sensitive color combinations where inappropriate.
- Provide alternate accent colors when needed.
- Maintain momo's warm neon identity while adapting respectfully.

### 10.2 Symbol Use

Use universal or BunnyEra-original symbols first.

Allowed:

- Light
- Bridge
- Signal
- Heart-like warmth, used subtly
- Digital garden
- Memory lantern
- Soft rabbit-inspired motifs

Use cultural symbols only when:

- The context is appropriate.
- The meaning is understood.
- The design is respectful.
- The symbol is not sacred, restricted, or misused.

### 10.3 Layout Adaptation

Support:

- Left-to-right and right-to-left layouts
- Localized spacing preferences
- Local reading patterns
- Different screen density expectations
- Market-specific UI comfort levels

Recommended baseline:

- Chrome 90+
- Edge 90+
- Firefox 88+

For production web apps, confirm compatibility through actual testing.
